# Changelog

## Unreleased

- ZTE模块扩展为任意目标热膨胀曲线反向设计：支持恒定目标、文本控制点和CSV/TXT导入的分段线性 `alpha_target(T)`；ROM、Turner、Kerner及全库向量化排名统一按目标曲线最小二乘优化，图表和PDF显示随温度变化的目标容差带；
- ZTE全库筛选新增工程约束：可指定组合必须包含或排除的元素，要求质量分数可计算及两相K/G字段完整，并限制密度、体积模量和剪切模量失配比；约束在NumPy批量排名前统一执行，结果表、Pareto浮窗、项目保存和科研PDF均保留工程指标与筛选上下文；
- ZTE页面新增手动配对与全库筛选双模式；批量引擎使用NumPy严格评估最多1,239,685个PTE/NTE组合，支持温区、模型、Kerner基体、材料限定词和体积分数约束，并按覆盖率、最长连续温区与误差排名；
- 全库候选可一键回填到手动设计，继续计算密度、质量分数、完整混合曲线和模型适用性；Agent新增严格全库ZTE组合筛选工具；
- ZTE全库筛选增加结果工作区：最多勾选6组候选叠加真实混合曲线，在NTE体积分数—RMS Pareto景观中按ZTE覆盖率着色，并导出CSV、JSON和离线PDF报告；
- ZTE筛选条件、完整排名与已选候选可保存到项目内 `workspace.sqlite`，支持跨刷新加载和删除，且曲线对比与PDF均由后端重新读取目录库计算；
- ZTE Pareto多目标景观支持鼠标悬停浮窗，直接显示材料组合、排名、配比、覆盖率、RMS及连续满足温区；
- ZTE Pareto点、排名表高亮和候选勾选实现双向联动；单击散点可选择候选并定位表格，表格悬停会在景观中强调对应点；
- 新增可分享链接的ZTE候选详情工作区，集中展示PTE/NTE两相可旋转三维结构、原始QHA曲线、ROM/Turner/Kerner三模型曲线、配比与误差对照、公式和满足温区；
- 候选详情支持下载两相POSCAR、`thermal_expansion.dat`、单相曲线PDF，以及候选曲线CSV、结果JSON和包含三模型对照的科研PDF；
- ZTE复合设计新增Kerner基体约束模型，允许选择PTE或NTE连续基体，并读取两相剪切模量以描述近球形分散颗粒的弹性约束；
- ZTE复合设计增加线性ROM与Turner体积模量加权模型对比，统一在用户指定温区的等间隔采样点上优化固定配比；
- 从数据库内POSCAR/CIF自动计算两相密度，并同时输出NTE体积分数和质量分数；
- 增加RMS、平均绝对偏差、最大偏差、ZTE温区覆盖率及连续满足区间，并允许配置目标容差和温度步长；
- ZTE曲线高亮目标温区和目标α容差带，浓度滑块按当前模型实时更新体积/质量分数与误差指标；
- 材料数据库增加 G、Ẽ、ξ、CTE 全库排序，CTE 区间筛选和 50/100/200 条显示控制；
- 增加最多 4 个材料的本地持久化收藏，支持属性表和真实 QHA α(T) 曲线叠加对比；
- 材料详情增加数据版本、源文件 SHA256、ALIGNN 剪切模量说明、Ẽ 公式和 QHA 曲线来源；
- Agent 重构为最多12步的自主模型—工具循环，能够根据工具结果修正SQL和后续计划；
- 增加目录库结构发现和受限只读SQLite查询，模型可自行完成任意温度插值、全库排名、组合筛选和统计，不再依赖每种问题的专用函数；
- 旧材料搜索、比较和温度排名工具保留为兼容接口，但不再暴露给模型占用工具上下文；
- fast、elastic、qha统一通过 `request_calculation_task` 创建持久化审批，已有待确认请求时禁止重复提交；
- 页面刷新后自动恢复待审批任务卡，批准后继续进入原后台任务与进度轮询链；
- 将结构预测工作区标记为实验性扩展，明确当前版本以数据库查询和分析为主。
- 材料详情支持直接下载数据库内POSCAR、按 `1/K` 重建的完整 `thermal_expansion.dat` 和离线曲线PDF；
- 收藏组合可保存为命名分析项目并跨刷新恢复，支持载入、更新和删除；
- 材料对比支持导出CSV、JSON、独立HTML报告和包含属性表/QHA曲线的PDF报告。

## 0.10.0 - 2026-07-14

- 按论文定义统一键合模量 `E_tilde = U_V/n = 160.21766208*abs(E_coh)/(AAV*avg_cn)`，数据库列表、详情、景观和Agent不再使用不一致的历史字段；
- 源码启动缺少目录库时，可从公开GitHub Release自动下载便携包、验证双重SHA256并原子安装SQLite目录库；
- 增加Windows、macOS、Linux三系统GitHub Actions，自动安装公开目录库、运行测试和验证无密钥便携包；
- 增加“关于软件”工作区，集中展示软件版本、著作权人/开发单位、数据版本、科学定义、技术组成和源码仓库；
- 窄屏设备默认收起右下角Agent，避免对话窗口遮挡数据库和关于页面主体内容；
- 统一Python包、FastAPI、网页静态资源和便携包版本为0.10.0。

## 0.9.0 - 2026-07-13

- 增加 Windows、macOS、Linux 共用的项目内便携运行环境和启动器；
- 增加默认无密钥发布包与显式合作者私有包构建流程；
- 移除源码中的开发机计算绝对路径，改用项目内可选 `compute.env`；
- 材料详情读取数据库内真实 POSCAR，使用 pymatgen CrystalNN 构建跨晶胞周期键合图，并由本地 3Dmol.js 显示可旋转、缩放和平移的三维晶体结构；
- 增加球棍、空间填充、键线、晶胞、周期邻居、超胞、视角重置、全屏和图片保存控制；
- 参考 Materials Project 的 VESTA 配色、统一原子半径、周期镜像和竖向晶胞表达，并将三维结构与真实 QHA 热膨胀曲线等宽等高并排展示；
- 将固定 CSS 坐标轴替换为独立 WebGL 三维坐标轴，仅同步主晶体的旋转四元数，并支持随重置视角恢复；
- 将三维坐标轴的 X/Y/Z 改为高清 HTML 标签，保持与 WebGL 箭头端点同步旋转并改善小尺寸显示清晰度；
- Fig. 1d 散点与当前材料星标按绘图区尺寸保持固定视觉比例平滑缩放，并同步调整悬停命中范围；
- 参考 Materials Project 重构数据库入口：将四个统计卡压缩为单条数据库摘要，并增加可折叠元素周期表；
- 元素周期表显示每种元素在6701个材料中的覆盖数量，支持“包含所选元素”和“仅含所选元素”的真实 SQLite 联合筛选；
- 将单页长页面拆分为材料数据库、结构预测、热膨胀景观和 ZTE 复合设计四个独立工作区路由，并保留全局 Agent；
- 元素周期表改为材料数据库工作区的常驻筛选组件，取消展开开关；
- 增加跨工作区“当前研究材料”上下文条、景观定位入口和状态恢复，让数据库材料及结构预测结果在多页面间保持连续分析体验；
- 三维渲染库和许可证随便携包离线分发，pymatgen/CrystalNN 作为项目内 Python 环境依赖安装。
