# 数据契约 V1

## 核心实体

- `materials`：跨数据版本稳定的材料标识、化学式和外部数据库编号；
- `dataset_releases`：不可变的数据集版本；
- `dataset_memberships`：材料与数据集版本的对应关系；
- `structures`：结构格式、内容及SHA256；
- `material_properties`：属性名称、数值或文本、单位及所属版本；
- `data_quality_flags`：异常代码、严重级别、原始值和处理说明；
- `calculation_jobs`：工作流、模型、参数、状态和结果；
- `composite_designs`：PTE/NTE组合、模型、配比和预测结果。

## 6702条数据的字段

首版共26个字段：材料标识、POSCAR及24项材料属性。数值属性采用原文件中的单位，单位映射在`catalog/importer.py`中显式维护。

导入过程不静默修改异常值。当前快照会标记1条非正体积模量记录和25条孔隙率超出`[0, 1]`的记录，由后续质量控制流程决定能否进入SBR、QHA或复合设计。

## 可追溯要求

每个正式计算结果后续至少需要保存：

- 输入结构SHA256；
- 模型名称、版本和权重SHA256；
- 全部计算参数；
- 软件依赖版本；
- 运行状态、开始和结束时间；
- 结果文件路径及SHA256；
- 质量标记和失败原因。
